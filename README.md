# Equilibr

`Equilibr` — это Django-приложение для работы с термодинамическими данными, расчёта химического равновесия и последующего анализа результатов в интерфейсе `ICP Stat`.

Проект объединяет три рабочих модуля:

- `ivtanthermo` — каталог веществ и термодинамических данных;
- `equilibrium` — расчёт равновесия по данным из БД;
- `icp_stat` — анализ и визуализация импортированных концентраций ионов.

По умолчанию проект работает на SQLite и уже содержит готовую локальную базу `db.sqlite3`.

## Возможности

- просмотр базы веществ и их термодинамических карточек;
- read-only админка для просмотра данных;
- расчёт равновесия по встроенной базе веществ;
- расчёт по диапазону температур;
- учёт давления;
- ввод исходного состава в молях или массах;
- сохранение результатов расчётов;
- построение графиков на `Plotly`;
- экспорт расчётных таблиц в `CSV`;
- добавление пользовательских веществ прямо из веб-интерфейса;
- импорт ионных концентраций из `Equilibr` в `ICP Stat`.

## Структура проекта

```text
Equilibr/
├─ equilibr/          # настройки Django-проекта
├─ equilibrium/       # расчёт равновесия
├─ icp_stat/          # обработка и визуализация ICP данных
├─ ivtanthermo/       # каталог веществ и термодинамическая БД
├─ templates/         # общие шаблоны
├─ src/               # исходный Julia-код и связанные материалы

├─ db.test.sqlite3    # короткая тестовая база
```

## Основные URL

- `/` — список веществ;
- `/substance/thermo/<id>/` — карточка вещества;
- `/equilibrium/` — расчёт равновесия;
- `/icp-stat/` — анализ результатов для ICP;
- `/admin/` — админка Django.

## Быстрый запуск

Ниже минимальный рабочий сценарий для локального запуска на встроенной SQLite-базе.

### 1. Создать и активировать виртуальное окружение

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

### 2. Установить зависимости

```powershell
pip install -r requirements.txt
```

### 3. Применить миграции

```powershell
python manage.py migrate
```

### 4. Запустить сервер

```powershell
python manage.py runserver
```

После запуска приложение будет доступно по адресу:

```text
http://127.0.0.1:8000/
```

## Пример запуска с нуля

## База данных

### SQLite по умолчанию

Проект по умолчанию использует:

- `db.sqlite3` — основную рабочую базу;
- `db.test.sqlite3` — короткую базу для быстрых тестов и отладки.

В `equilibr/settings.py` SQLite является стандартным backend, если явно не включён Postgres.

Можно указать альтернативный файл SQLite через переменную окружения:

```powershell
$env:SQLITE_NAME = "db.test.sqlite3"
python manage.py runserver
```

### PostgreSQL опционально

Если нужно работать с PostgreSQL-источником, проект поддерживает запуск через `docker-compose`.

Запуск:

```powershell
docker compose up -d
```

После этого можно переключить Django на Postgres:

```powershell
$env:USE_POSTGRES = "1"
python manage.py runserver
```

По умолчанию используются параметры:

- `POSTGRES_DB=ivtanthermo`
- `POSTGRES_USER=ivtanthermo`
- `POSTGRES_PASSWORD=ivtanthermo`
- `POSTGRES_HOST=127.0.0.1`
- `POSTGRES_PORT=5433`

## Приложения

### `ivtanthermo`

Модуль каталога веществ и термодинамических данных.

Что умеет:

- показывает список веществ;
- открывает детальную карточку вещества;
- отображает свойства, коэффициенты, графики и библиографию;
- использует русские названия, если они есть в базе.

### `equilibrium`

Модуль расчёта равновесия.

Что умеет:

- считать равновесие по данным из БД;
- считать серию по температуре;
- учитывать давление;
- принимать мольный или массовый исходный состав;
- сохранять результаты;
- экспортировать результаты в `CSV`;
- строить графики на `Plotly`;
- добавлять пользовательские вещества с собственными коэффициентами.

Особенности:

- ручной `.dat`-ввод оставлен в интерфейсе как скрываемый служебный режим;
- пользовательские вещества сохраняются локально в БД и доступны в последующих расчётах;
- для массового ввода расчёт сначала переводит подачу в моли через молярную массу, затем считает равновесие.

### `icp_stat`

Модуль обработки результатов для ICP-анализа.

Что умеет:

- строить графики по загруженным данным;
- экспортировать результаты;
- импортировать ионные концентрации из сохранённых расчётов `Equilibr`;
- рассчитывать `m/z` как `mass / |charge|` только для ионных веществ.

## Пользовательские вещества

На странице `/equilibrium/` можно добавить собственное вещество без правки кода.

Для этого задаются:

- `label`;
- отображаемое имя;
- фаза;
- молярная масса;
- `dfh0`;
- температурный диапазон;
- элементный состав;
- 7 коэффициентов для расчёта термодинамической функции.

После сохранения вещество:

- попадает в локальную БД;
- становится доступным в подборе исходных веществ;
- участвует в расчётах наравне со штатным каталогом.

## Полезные команды

### Прогон тестов

Все основные тесты:

```powershell
python manage.py test equilibrium icp_stat ivtanthermo
```

Только расчёт равновесия:

```powershell
python manage.py test equilibrium
```

### Синхронизация зарядов веществ

Проект хранит корректный заряд вещества на уровне `Substance`, вычисляя его из `label`.

Команда пересинхронизации:

```powershell
python manage.py sync_substance_charges
```

### Пересборка SQLite-каталога

Из исходного источника данных можно собрать новую SQLite-базу.

Полная база:

```powershell
$env:USE_POSTGRES = "1"
python manage.py build_sqlite_catalog --output db.sqlite3 --subset full --force
```

Короткая тестовая база:

```powershell
$env:USE_POSTGRES = "1"
python manage.py build_sqlite_catalog --output db.test.sqlite3 --subset test --force
```

## Пример рабочего сценария

1. Открыть каталог веществ на `/`.
2. Перейти в `/equilibrium/`.
3. Выбрать исходные вещества из БД.
4. Задать диапазон температур и давление.
5. Выполнить расчёт.
6. Скачать `CSV` с результатами.
7. Сохранить расчёт.
8. Перейти в `/icp-stat/` и импортировать сохранённый результат из `Equilibr`.

## Замечания по данным

- В тестовом окружении SQLite не поддерживает `db_comment` у legacy-моделей. Django показывает предупреждения, но на работу приложения это не влияет.
- Часть legacy-моделей `ivtanthermo` использует существующие таблицы и данные каталога.
- Для длительных расчётов интерфейс показывает экран загрузки, но сам расчёт пока выполняется синхронно в HTTP-запросе.

## Что лежит в `src`

Папка `src` содержит исходную реализацию расчёта на Julia, на базе которой перенесён функционал в Django-приложение `equilibrium`.
